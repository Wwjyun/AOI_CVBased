# AOI CV Based 週報

日期：2026-07-16（星期四）至 2026-07-20（星期一，截至目前）

本週工作由 GPU 架構建置進一步推進到可驗證、可觀測與可操作的階段。主要完成共用前處理與 persistent CUDA 執行架構、RTX 3090 驗證工具鏈、CPU 與打包 fallback 驗證、GUI 工作流程改善，以及 Detector 401 端到端效能分析工具。期間共完成 48 筆提交，異動 78 個檔案，新增 11,694 行、刪除 1,434 行。

## 本週完成事項

### 1. CPU／CUDA 共用架構與正確性

- 建立 immutable `PreprocessPlan` cache、tensor contract 與 backend capability report，讓 CPU、CUDA 與 fallback 路徑可被一致追蹤。
- 將 Detector 401、401-1 與 900 遷移到 shared preprocess plan；Detector 900 支援 CPU DAG／multi-output 執行。
- 建立 generic persistent CUDA linear plan、DAG multi-output ABI、resident image／device ROI 與 VRAM-aware ROI batch，降低重複 H2D、配置及同步成本。
- 統一 `cpu`、`auto`、`cuda` 模式語意；CUDA 不支援或失敗時，依設定完整重跑 CPU，避免混用部分 GPU 中間結果。
- 補強 edge cases、舊 DLL、fake DLL、context failure、OOM recovery、shape/channel/parameter 變更等測試與 metadata。

### 2. GPU 效能觀測與 RTX 驗證準備

- 新增 CUDA event phase timings，分離 H2D、kernel、D2D、D2H、Morphology、Gaussian、Adaptive Mean 等耗時。
- 建立可重現 benchmark schema，包含 cold／warm、median／P95、VRAM、allocation、driver／toolkit／GPU 與 commit 資訊。
- 增加 64²～1024² crossover matrix、Morphology iterations 1／2／4／8、context reuse、100／1000 次 stress 與 48 小時 heartbeat／P95 regression gate。
- 建立 production acceptance manifest 與 RTX workflow input，可驗證五份 production recipes 的 PASS／NG、座標、缺陷資訊與 CPU／GPU 等價性。
- 加入可選 Nsight Systems capture；runner 有 `nsys` 時保存 report，否則明確記錄 skip 原因。

### 3. Detector 401 專項 profiling

- 新增 full-image Detector 401 profiler，分離 template matching、ROI generation、detector、pipeline-before-report、reporting、end-to-end 與 GUI user-wait 範圍。
- 新增離線 profile analyzer，可檢查 PASS／NG、座標、fallback gate、2 秒目標及 3.3 秒上限，並依證據排序 launch／ROI、同步、Morphology、D2H、Gaussian、Adaptive Mean、CPU contours 等瓶頸。
- GUI 單張檢測開始重用 `GpuExecutionSession`；相同 recipe 可重用 context，recipe 檔案變更時失效，視窗關閉時安全釋放。
- 優化 Detector 401-2 white-ratio 統計，512² synthetic microbenchmark median 由 0.0343 ms 降至 0.0151 ms，且維持 count、ratio、順序與 metadata 等價。

### 4. GUI 與操作流程

- 統一 TopBar 全域進度、步驟細節與短事件狀態層級，新增實際 backend chip 與 CPU fallback inline notice。
- 改善 Results NG 鍵盤導覽、bbox 聚焦、Recipe dirty／validation、QSettings 工作環境回復，以及 Batch／Monitor model-view 篩選。
- 對大型 scatter 資料採 deterministic sampling，降低介面負擔並維持可重現顯示。
- 新增 OOP 權限管理：啟動固定為 OP；切換工程／管理模式需分別輸入密碼，取消或輸入錯誤時維持原模式。

### 5. CPU 效能、報表與產線可追溯性

- 完成 recipe cache、opt-in CPU tile 平行、OpenCV thread budget、週期 GC、NG tile 平行寫檔與 overlay 輸出優化。
- 補上 strict detector schema、recipe／build SHA-256、Git commit provenance、NG dataset sidecar 與合成 golden cases。
- 建立 Python 3.13 Windows dependency lock、Windows CPU CI、RTX 3090 self-hosted workflow 與 weekly package smoke。
- 修正 Windows CLI smoke exit code：PASS=0、NG=2，其餘未預期錯誤會正確使 CI 失敗。

### 6. Windows 打包與 fallback 驗證

- 修正 `build_exe.ps1` 覆寫受版控 spec 的問題，改以固定 `VisionFlow AOI.spec` 建置。
- CPU-compatible PyInstaller EXE 已在無 NVIDIA GPU／無 CUDA DLL 環境完成 packaged smoke，涵蓋 bundled recipes、MainWindow、CPU-only pipeline、缺 DLL auto fallback 與 strict CUDA 明確失敗。
- CPU-only 與缺 DLL fallback 的 PASS／NG、tiles、defects、bbox 與 metadata 一致，且確認 GPU call count 為 0。

## 驗證成果

- 自動測試隨本週功能由 119 項增加至 139 項，最新完整測試全數通過。
- `compileall`、CUDA source preflight、CLI synthetic smoke、GUI offscreen smoke 與 `git diff --check` 均通過。
- CPU-compatible packaged smoke 實際 exit 0；已確認五份 bundled recipes 可載入。
- Windows CI 已完成；RTX workflow 與 production benchmark 工具已就緒，但目前 repository 尚無可用的 self-hosted RTX runner。

## 問題與風險

- 目前開發機沒有可用的 `nvcc`、MSVC `cl`、NVIDIA GPU／CUDA device，因此新增 CUDA 原始碼尚未在 RTX 3090 重建 DLL 與實際執行。
- 尚缺固定且可追溯的真實 AOI production 影像；五份 recipe 各一組 PASS／NG、CPU／GPU binary agreement 與完整 defect metadata 驗收仍待進行。
- Detector 401 的真實 GPU cold／warm／GUI 連續測試尚無數據，目前不能宣稱已達 warm median 3.3 秒門檻或 2 秒目標。
- CUDA error／OOM、VRAM 穩定性、1000 張長時間測試、有 GPU 的打包版本及 ABI mismatch 仍需目標硬體實測。
